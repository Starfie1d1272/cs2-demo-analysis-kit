## 3. RivalHub 现状架构（调研结论 · 关键）

```
ZIP →[demo-import.ts]→ parseDemoPackage(zipBuffer)  // 用 cs2-demo-format/parser，与 kit 同源
      → parsed = { manifest, files:{ players,rounds,playerStats,playerEconomies,kills,
                                      damages,blinds,bombs,clutches,grenades,shots,positions } }
      → 打散 insert 进 Postgres 多张表（drizzle）
展示时 →[demo-detail.ts]→ 读 DB 行 → 重组 EconomyRow[]/DemoPlayerStatRow[]
      → 自己派生（economy-series.ts / to-rr-indicators.ts / economy-conversion.ts）
```

**核心错配**：kit 的纯函数吃 `DemoPackage`（解析后的 ZIP），RivalHub 展示层喂的是 **Postgres 行**。
**但** RivalHub 在 import 那一刻手里就有完整 `parsed`，只是用完拆库了 → 这是接 kit 的天然入口。

**喂 kit 的适配器（已验证可行，shape 差异极小）**：
```ts
// cs2-demo-format/parser 出 { manifest, files }；kit normalizeDemoPackage 要 manifest + 顶层文件数组
import { normalizeDemoPackage, computeAccountRatingsV2, buildEconomy, buildEconomyConversion } from "@cs2dak/core";
const pkg = normalizeDemoPackage({ manifest: parsed.manifest, ...parsed.files });
const ratings = computeAccountRatingsV2(pkg);
const economy = buildEconomy(pkg);          // EconomyPoint[]
const conversion = buildEconomyConversion(economy);
```
