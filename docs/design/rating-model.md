# RR 评分模型设计（面向 CS2 玩家）

> 这篇是**整体方案**，目标是：有一定游戏理解的 CS2 玩家，不用懂代码也能看懂"这个分是怎么来的、为什么这么设计"。
> 公式里的系数都是**模型里真实在用的数值**。技术实现细节在 [rr-v1.md](rr-v1.md)（HLTV 基线）、[prism.md](prism.md)（八维风格）、[cohort.md](cohort.md)（跨场聚合），公式代码归属在 [`@rivalhub/rival-rating`](https://github.com/Starfie1d1272/rival-rating)。

---

## 一句话

RR 给你一场（或一个赛季）的发挥打一个分，**1.0 = 标准水平**，越高越好。思路和 HLTV Rating 2.0 类似——但 HLTV 基本只看你"枪打得好不好"，RR 额外把**补枪、残局、道具、目标**这些团队贡献单独记账，因为我们认为**市场（HLTV/ratingPro）低估了团队价值**。

## 别把三个东西搞混

| 名字 | 回答什么 | 一句话 |
|---|---|---|
| **RR v1** | 这场数据产出如何 | HLTV 2.0 逆向公式，纯盒分基线，[详见 rr-v1.md](rr-v1.md) |
| **RR v2（本文）** | 你这场/这赛季的贡献值多少 | 五账户模型，**这是主角** |
| **PRISM** | 你是什么风格 | 八维画像，不分高低，[详见 prism.md](prism.md) |

下面只讲 **RR v2**。

---

## 核心思想：把表现拆成五个"账户"

不像 HLTV 把所有东西揉进一个公式，RR v2 把你的贡献分门别类记进五个账户，每个账户管一类游戏行为，最后按重要性加权汇总。**这样你能看到自己的分到底靠什么挣的。**

| 账户 | 奖励什么游戏行为 | 权重 | 为什么是这个权重 |
|---|---|---|---|
| **Combat 枪火** | 杀人、伤害、不送、首杀、多杀 | **1.0** | 赢比赛最硬的还是枪，主干 |
| **Trade 补枪** | 补枪、被有效换、不白给 | 0.5 | 团队配合的核心 |
| **Clutch 残局** | 该赢的残局赢下、能偷就偷 | 0.4 | 关键分，但样本少 |
| **Utility 道具** | 闪好闪、道具伤害、不闪队友 | 0.3 | 鼓励大家多扔道具配合 |
| **Objective 目标** | 下包、拆包、下包后赢 | **0.1** | 故意压低，下包者常只是战术末端，噪声大 |

下面逐个账户给**真实公式**（都按"每回合"计算）。

### 🔫 Combat 枪火（权重 1.0，主干）

```
枪火原始分 = 0.55×每回合击杀
           − 0.35×每回合死亡
           + 0.003×每回合有效伤害
           + 0.3 ×每回合(首杀 − 首死)
           + 0.4 ×每回合多杀回合
           + 0.05×每回合穿墙击杀
```

**人话**：主要就是你的 KPR（每回合击杀）、DPR（每回合死亡）、ADR（每回合伤害），外加首杀和多杀的小奖励。死亡是扣分的。这部分是整个评分的大头。

**还有一个"打架背景"乘子**——同样一个击杀，背景不同价值不同，只作用在击杀项上：

| 背景 | 乘子 | 意思 |
|---|---|---|
| 以弱打强（装备差 >$1000） | ×1.35 | 穷鬼局/起手枪杀满编，加分 |
| 以强凌弱（装备强 >$1000） | ×0.8 | 顺风局补刀，降权 |
| 人数劣势击杀 | ×1.2 | 1v2 里你杀人比 5v2 值钱 |
| 人数优势击杀 | ×0.9 | 人多打人少的击杀降权 |

> 这个乘子需要导出数据里有"击杀时双方装备/人数"才生效；没有就当 1.0（不影响）。

### 🤝 Trade 补枪（权重 0.5）

```
补枪原始分 = 0.6 ×每回合补枪          （队友倒了你马上补回来）
           + 0.3 ×每回合被成功换       （你死了队友能换回来 = 你死得有价值）
           − 0.25×每回合白给死亡        （没人能换的孤儿死亡）
```

**人话**：奖励"队友倒下你立刻补"和"你即使死了，队友也能用你的死换一个回来"（说明你站位有配合、没乱送）。惩罚"孤零零冲出去白给、队友够都够不到"。

> ⚠️ 已知短板：**自由人/lurker 单独行动，他的击杀死亡不发生在补枪网里**，所以这个账户会系统性低估自由人。未来的"地图控制账户"就是为了补这个缺口（见下方路线）。

### 🎯 Clutch 残局（权重 0.4）

```
残局净超额 = Σ ( 实际赢的残局数 − 遇到的次数 × 该局面期望胜率 )

期望胜率：1v1 = 0.5   1v2 = 0.25   1v3 = 0.1   1v4 = 0.04   1v5 = 0.01
```

**人话**：不是简单数你赢了几个残局，而是看你**比"正常人该赢的"多赢了多少**。
- 赢一个 1v3：+(1 − 0.1) = **+0.9**（远超期望，大赚）
- 输一个 1v3：+(0 − 0.1) = **−0.1**（几乎不扣，因为 1v3 本来就该输）

所以它**奖励"该赢的赢、能偷的偷"，但不为难残局打不好的人**——输难局几乎没惩罚。

### 💡 Utility 道具（权重 0.3）

```
道具原始分 = 0.4  ×每回合闪光助攻      （闪到人，队友借光击杀）
           + 0.05 ×每回合致盲敌人秒数
           − 0.05 ×每回合致盲队友秒数   （闪到自己人，扣）
           + 0.004×每回合道具伤害       （燃烧弹/手雷伤害）
```

**人话**：扔得好的道具该得分——闪能让队友白嫖击杀、燃烧逼人、手雷磨血。闪到队友扣分。**这个账户的存在就是为了鼓励大家多买多扔道具配合，而不是只顾自己枪。**

### 🧨 Objective 目标（权重 0.1，故意压低）

```
目标原始分 = 0.2×每回合下包 + 0.3×每回合拆包 + 0.2×每回合(下包后赢下)
```

**人话**：下包拆包给一点分，但权重很低。因为"谁下的包"往往只是战术链末端（道具铺好、人都杀完了，随便谁去下），噪声大，不该给太多分。

---

## 五个账户怎么合成一个 RR

```
RR 原始分 = 0.5 (底分)
          + 1.0×枪火 + 0.5×补枪 + 0.4×残局 + 0.3×道具 + 0.1×目标
```

然后做一步**归一化**，把它变成 1.0 为中心的好懂刻度。**归一化怎么做，正是当前方案和未来方案的核心区别**，也是这套系统下一步最重要的升级。

---

## 当前方案 vs 未来方案（1.0 到底代表谁）

### 当前：赛季相对（cohort z-score）

- **1.0 = 被分析的这批选手的平均水平。**
- 五账户先在这批人里算"你领先几个标准差（σ）"，combat 当主干，其余账户**残差化**（把和枪法重合的部分剔掉，只留"超出你枪法水平的额外团队贡献"，避免一个枪好的人每项都白拿分），再缩放到和 HLTV 差不多的刻度。
- **问题（这就是要改的原因）**：
  - 换一批人，同样的发挥分数就变 → **不可移植**。
  - 单独一个 demo 没法绝对评分（cohort 就这 10 个人，全是互相相对）。
  - 1.0 是浮动的，说不清是"职业平均"还是"天梯平均"。

### 未来：职业基准固定（冻结一条标准曲线）★ 方向

- **1.0 = 职业平均水平**——一把**固定的尺子**，不随被测的人变。
- 做法：从**大量职业 demo** 里把每个账户的分布算出来，冻结成一条**标准曲线**（固定参数），写死进配置。
- 之后**任意一个 demo（哪怕随便一个天梯局）丢进去，对着这把固定尺子量** → 得到绝对分、可移植、单 demo 就能用，不需要 cohort。
- 残差化的"枪法→账户"关系也一并从职业数据里拟合、冻结。
- **额外好处**：用"曲线"而不是裸 z-score，尾部天然饱和——道具打爆表的人（比如 +3.8σ）不会再一柱擎天单独刷高分，曲线自动把极端值压住。

#### 关于"天梯均值会掉到 0.8–0.9"

拿职业当标杆，普通天梯玩家的补枪/道具效率自然只到职业的 80–90%，**所以天梯均值掉到 0.8-0.9 是有意为之、也是正确的**——它就是在说"离职业还有多远"，反而能促进大家多补枪多扔道具。

这是**展示层的问题，不是模型的问题**。模型永远给绝对的职业基准分；展示时可以叠一层友好化，比如"你在你段位里的百分位"或"职业达成度 85%"。**模型一套、皮肤多套，不为了好看去污染模型。**

---

## 为什么要换

一句话：现在所有的分都"绑死在当前这批人身上"、没法复用。我们想要的是**一个像 HLTV Rating 一样、拿出任何一个 demo 都能算出标准分的固定公式**，而且因为账户拆得清楚、系数都摆在台面上，**任何玩家都能看懂自己的分是怎么挣的**。职业基准 = 标杆，鼓励所有人向"多补枪、多给道具配合"看齐。

## 后续工作方向（路线）

详细阶段见 [rr-roadmap.md](../rr-roadmap.md)，与本文相关的核心几步：

1. **归一化重设计**：把"赛季相对 z-score"换成"职业基准固定曲线"——接口先用现有 55 场赛事数据跑通，再换数据源。详见 [pro-baseline.md](pro-baseline.md)。
2. **搭职业 demo 语料**：100–200 张 map 起步，数据源为 **HLTV 职业 GOTV demo**（CS2 MR12 时代，Tier-1 LAN）；FACEIT 高分段不适合作职业基准尺（无法冻结"职业平均 = 1.0"），可作补充但不做主干。
3. **道具/全账户尾部饱和**：被未来的曲线方案自动包含（当前可先单独做 σ 封顶）。
4. **地图控制账户**：基于每秒位置 + 地图区域划分，奖励"独自拿/守空间"，**补自由人补枪被低估的缺口**。先做复盘/风格轴，暂不进 RR 总分。
5. **身份合并**：已支持（外接 userId 把多个 steam 号归并成一个人）。

## 校准状态（务必知情）

当前权重是**凭游戏理解定的先验**，不是从数据回归出来的。已用 2026 NJU Rivals 的 55 场 + OCR 的 ratingPro/WE 做过验证：**整体评分高度由枪火主导，补枪/残局/道具的"独立增量"对市场分几乎没有预测力**——也就是说，市场（ratingPro/WE）本身就不怎么认团队价值。所以 RR v2 强调团队 = **刻意的价值主张**，目前没有外部数据背书，对错要靠看比赛和大家的主观判断。**别用它给选手发正式定级，先当参考和讨论用。**
